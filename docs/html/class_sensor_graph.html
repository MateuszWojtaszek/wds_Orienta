Opis szczegółowy

Dziedziczy po QChartView i umożliwia dynamiczne rysowanie danych z trzech kanałów (np. osie X, Y, Z akcelerometru) w czasie rzeczywistym. Wykres automatycznie przesuwa się wraz z napływem nowych danych, a najstarsze punkty są usuwane po przekroczeniu zdefiniowanego limitu próbek.

Nota
Wykorzystuje bibliotekę Qt Charts.

Inicjalizuje obiekt QChart, tworzy i konfiguruje trzy serie danych (dla osi X, Y, Z), ustawia osie wykresu (czasową X i wartości Y), konfiguruje legendę oraz ustawia tytuł wykresu na podstawie klucza titleKey. Definiuje również początkowy zakres osi Y oraz domyślną liczbę widocznych próbek.

Parametry

[in]minYMinimalna wartość początkowa dla osi Y.
[in]maxYMaksymalna wartość początkowa dla osi Y.
[in]parentOpcjonalny wskaźnik na widget nadrzędny w hierarchii Qt. Domyślnie nullptr.

◆ addData()

void SensorGraph::addData ( const QVector< int > & axisValuesToAdd)

Oczekuje wektora QVector<int> zawierającego dokładnie 3 wartości, które są dodawane do odpowiednich serii danych (X, Y, Z) z bieżącym indeksem próbki jako współrzędną X. Jeśli liczba punktów w serii przekroczy defaultSampleCount, najstarszy punkt jest usuwany. Zakres osi X jest okresowo aktualizowany, aby przesuwać "okno" widoku.

Parametry
[in]axisValuesToAddWektor zawierający 3 liczby całkowite (wartość dla serii X, Y, Z).
Nota
Jeśli rozmiar wektora axisValuesToAdd jest inny niż 3, dane nie zostaną dodane.

◆ setSampleCount()

void SensorGraph::setSampleCount ( int sampleCount)

Określa rozmiar "okna czasowego" wykresu. Po dodaniu nowych danych, jeśli liczba punktów przekracza ten limit, najstarsze punkty są usuwane z każdej serii. Minimalna dozwolona wartość to 10. Zakres osi X jest automatycznie dostosowywany do nowej liczby próbek.

Parametry
[in]sampleCountNowa maksymalna liczba widocznych próbek (min. 10). Zmieniono typ na int dla prostoty, const int& jest rzadko potrzebne dla typów podstawowych.
