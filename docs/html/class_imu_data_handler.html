Opis szczegółowy

Klasa ta integruje odbieranie danych sensorów, ich prezentację w formie numerycznej (paski postępu) i graficznej (wykresy czasowe), a także renderowanie sceny 3D pokazującej orientację urządzenia oraz kierunek wskazywany przez kompas. Jest to główny widget odpowiedzialny za interfejs związany z danymi IMU.

◆ setRange()

void ImuDataHandler::setRange ( int minVal,
int maxVal )

Konfiguruje minimalne i maksymalne wartości wyświetlane na osi Y wykresów oraz zakresy dla pasków postępu, aby dopasować je do oczekiwanych wartości z sensorów.

Nota
Aktualnie funkcja ustawia predefiniowane, sztywne zakresy dla każdego typu sensora, ignorując przekazane parametry minVal i maxVal. Zakresy: Akcelerometr (-4000, 4000), Żyroskop (-250, 250), Magnetometr (-1600, 1600).
Parametry
minVal[in] Minimalna wartość zakresu (obecnie ignorowana).
maxVal[in] Maksymalna wartość zakresu (obecnie ignorowana).

◆ setRotation()

void ImuDataHandler::setRotation ( float yaw,
float pitch,
float roll )

Konwertuje podane kąty odchylenia (yaw), pochylenia (pitch) i przechylenia (roll) na kwaternion reprezentujący orientację, a następnie stosuje tę transformację do modelu 3D płytki (boardTransform) w scenie Qt3D. Jeśli transformacja boardTransform nie została jeszcze zainicjalizowana (np. model się nie załadował), operacja nie ma efektu.

Parametry
yaw[in] Kąt odchylenia (obrót wokół osi Z) w stopniach.
pitch[in] Kąt pochylenia (obrót wokół osi X) w stopniach.
roll[in] Kąt przechylenia (obrót wokół osi Y) w stopniach.

◆ updateCompass()

void ImuDataHandler::updateCompass ( float heading)

Obraca strzałkę kompasu (compassTransform) wokół osi Z sceny 3D, aby wskazywała kierunek geograficzny odpowiadający podanemu kątowi heading. Kąt 0 stopni odpowiada Północy. Jeśli transformacja compassTransform nie została zainicjalizowana, operacja nie ma efektu.

Parametry

◆ updateData()

void ImuDataHandler::updateData ( const QVector< int > & acc,
const QVector< int > & gyro,
const QVector< int > & mag )

Odbiera najnowsze odczyty z sensorów IMU i przekazuje je do odpowiednich elementów interfejsu: aktualizuje wartości na paskach postępu (QProgressBar) oraz dodaje nowe punkty danych do wykresów czasowych (SensorGraph). Sprawdza rozmiar wektorów wejściowych (oczekiwany rozmiar to 3).

Parametry
acc[in] Wektor danych z akcelerometru [X, Y, Z] (jednostki: mg).
gyro[in] Wektor danych z żyroskopu [X, Y, Z] (jednostki: dps).
mag[in] Wektor danych z magnetometru [X, Y, Z] (jednostki: mG).
